# 個人 DNA 萃取範本

把這份範本複製到記事本，填好之後丟給 Cowork。Cowork 會用這些資料客製化工具。

---

## 範本 1 · 個人說明書

複製下面這段，把【】裡的填好：

```
# 個人說明書

## 我是誰
- 名字 / 暱稱:【例:Summer / 姐 / 老闆娘】
- 公開身份:【例:HCSXMME 私密社團主理人 / AI 講師】
- 年齡 / 階段:【例:38 歲新手媽 / 退休教授 / 大學生】
- 所在地:【例:台北 / 高雄 / 加拿大】

## 我的品牌
- 品牌名:【例:小櫻優選社】
- 品牌定位:【一句話講你的品牌做什麼,例:給講究質感的職場媽媽選好物】
- 創立故事:【1-2 句話,例:從自己生完小孩後到處找好物,後來開始幫朋友開團】

## 我的客群
- 主要年齡:【例:28-42 歲】
- 性別:【例:女性為主 / 不分性別 / 男性為主】
- 生活情境:【例:雙薪家庭新手媽 / 退休族 / 自由工作者】
- 共同特質:【例:預算中上、看重品質、不喜歡業配感太重】

## 我的核心優勢
- 我跟競爭者最大不同:【例:我自己會用過 1 個月才推,絕不做沒試過的商品】
- 客戶最常稱讚我:【例:選品準、客服快、文案有溫度】
- 我絕不做的事:【例:不點名競品、不推誇大療效、不做太低單價】
```

## 範本 2 · 職務說明書

```
# 職務說明書

## 我賣什麼
- 主要產品 / 服務類別:【例:美妝保養 / 親子用品 / AI 課程 / 心理諮詢】
- 單價區間:【例:NT$ 300-2000 / 課程 NT$ 5000-30000】
- 上架頻率:【例:每月 3-5 團 / 每週 1 篇分享】

## 通路與客人
- 主要通路:【例:FB 社團、IG、LINE 群、自己網站】
- LINE 客服 ID:【例:@hcsxmme,@yourlineid】
- 客人怎麼下單:【例:FB 留言喊單、LINE 私訊、網站結帳】

## 運作模式
- 結團週期:【例:每月 1 號開、月底結團】
- 到貨時間:【例:結團後 2-3 週 / 結團後一週】
- 售後處理:【例:LINE 客服處理,有換貨保證】
- 我每篇 FB 文必有的元素:【例:價格、規格清單、結團時間、喊單範例、LINE ID】

## 我的禁區
- 絕對不寫的話:【例:保證見效、絕對美白、X 天必瘦】
- 不能曝光的資訊:【例:廠商進貨價、客戶真實姓名、合約細節】
- 不點名的對手:【例:跨足同類別的另一家品牌】
```

## 範本 3 · 粉專備份（最快版本）

如果不想正式申請 FB 資料下載，最簡單做法：

1. 開個記事本（或 Google Docs）
2. 到你 FB 粉專或個人時間軸
3. 複製過去 6-12 個月**寫得最滿意的 20-50 篇文章**完整內文
4. 一篇一篇貼進記事本，每篇之間用 `===` 分隔
5. 存成 `粉專備份.txt`

格式範例：

```
===
冬天手乾到衣服都勾線?姐這條 Aesop 粉紅管擦三天就回不去!
[完整內文]
===
為什麼這款護手霜會成為妳化妝包的新寵?
[完整內文]
===
[第三篇...]
```

Cowork 會從這 20-50 篇文章自動萃取你的：
- 開頭句型（前 5 個典型）
- 口頭禪 top 20
- 常提到的家人朋友角色
- 句子長度模式
- 標點符號習慣
- 第三方背書方式

---

## 萃取完之後

把這三份檔案放在 Cowork 連接的資料夾裡（任何位置都行），然後照 [用Cowork客製化部署.md](用Cowork客製化部署.md) 操作。

Cowork 會：
1. 讀你的三份 DNA 素材
2. 寫出客製版 SYSTEM_PROMPT（裡面有你的口頭禪、寫作公式、禁區）
3. 改 generate-background.js 把客製內容塞進去
4. 部署到 Netlify
5. 給你網址

你就有屬於你自己的 AI 發文器了。
